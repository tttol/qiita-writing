# AWS Community Buildersとは？
よく聞かれる質問の一つが、「AWS Community Buildersとは一体何なのか？」という点です。
これは、AWSコミュニティの成長、学習、拡大に情熱を注ぐクラウド愛好家の集団です。
Amazon Builder Centerのコミュニティプログラムの一部として紹介されており、ウェイトリストへの登録も可能です。
CBsに選出されると、いくつかの大きなメリットがあります。
• CBsメンバー限定のSlack招待
    ◦ 世界中のCB達と交流することができます。
• SWAG（グッズ）
    ◦ リュック、タンブラー、パーカーなど、様々なSWAGが提供されます。

# CBsになるために何をしたか？ — 私のアウトプットへのスタンス
LTやコミュニティ活動で最もよく聞かれる質問が、「CBsになるために何をやりましたか？」というものです。この問いに対する私なりの考え方は、活動の原動力にもなっています。
結論から言うと、「CBsになること」を目標にして頑張るのは辛いと感じています。
これは、特定の目標達成のために、本来の目的とは関係なく活動を続けることが、精神的にも負担になる可能性があるためです。
私が好ましいと考えているのは、アウトプットを継続していたらいつの間にかCBsになっていたという形です。
私の「アウトプットに対するスタンス」は以下の通りです。
1. 自分が忘れないためにブログを書く、または登壇して記録に残す。
2. もし忘れてしまっても、後から見返すことができるようにしておく。
例えば、以前Qiitaに投稿したJVMのGarbage Collectionに関する記事 や、NeovimのLSP設定 なども、自分の知識を整理し、忘れないようにするためにアウトプットしたものです。
このように、純粋に技術的な興味や、自分の学習履歴を残すことを目的にアウトプットを続けていました。この「アウトプットの継続」こそが、結果的にCBs選出につながったのだと感じています。

# そして、2024年にCBsに
そして、アウトプットを続けていた結果、2024年にAWS Community Builderに選出されることができました。
Twitter（現X）でも選出されたことを報告しましたが、**「AWSゼロ冠の私ですが、この度AWS Community Builderに選出されました。」**と記載した通り、資格の有無に関わらず、コミュニティへの貢献やアウトプットが評価されたのだと理解しています。

# さいごに
改めてまとめになりますが、AWS Community Buildersを目指す上でも、技術者としての成長を目指す上でも、**「アウトプットを継続していたらいつの間にか目的地に到達していた」**という状態が、最も健全で好ましいと考えています。
この記事が、これからコミュニティ活動やアウトプットを始める方々の参考になれば幸いです。
